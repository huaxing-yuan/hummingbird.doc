Welcome

Hummingbird is an integrated environment for API Testing & Test Automation. It is an extensible, easy to use solution adapted to testers of all levels.

Getting Started

This documentation is designed to 4 types of users:

  • Testers who will use Hummingbird and Hummingbird CLI to run Manual and/or Automated Tests. In this topic, you will learn basic functinalities

  • Testers who have advanced knowledges about Web Services and is autonomy to create extensions by importing the Web Services Descriptions with Extension Designer.

  • Testers or Developers who have advanced knowledges about Web Service, Network Protocols and .NET programming skills who will create and custom extensions.

  • Testers or Ops who want to integrate automated tests into their CI/CD pipeline.

For different type of users the documentation is organized as:

  • Hummingbird User’s Manual

    You will understand the basic functionalities of Hummingbird: Run manual Tests with intelligent request editor, Mock services with service simulation, Manage test environments and variables, Create and run automated tests, Use assertions to validate automated tests, Analyze result with Statistics and Reports.

  • Extension Designer

    Focused on importing services from existing service description and configure the extension for other developers and testers. With Extension Designer, you can quickly import ordinary web services defined by WSDL and Swagger and start test or simulation.

  • Advanced Customization Guide

    The developer’s guide of the Hummingbird Test Framework and API references to give the ability of Testers and Developers to adapt the software to a specific usage. With this level of customization, you can use Hummingbird to test and automate your own specific web services, and even the specific network protocol.

    In additional to this development guide, we have created two open source extensions:

    Hummingbird.Extension.Messaging implements SMS and MMS (MM3, MM7) protocol used by telecommunication equipment: SMS is based on a customized TCP socket protocol, MM3 is based on SMTP protocol and MM7 is based on Soap Web services encoded in MToM. This extension is built to demonstrate how to implement new protocols to change productivity of the Dev and Test team. This extensions is built to give you an idea how you can make your own extension to meet special needs. It is strictly prohibited to use these extension for real testing activities in an enterprise environment.

    Hummingbird.Extension.Selenium gives you the ability to run Web based tests with using Selenium Web Driver.

  • DevOps Guide

    In this section, we will mainly discuss how to integrate the automated tests into DevOps, we will make examples for using Visual Studio Team Services and Jenkins.

The documentation and the Hummingbird software applies standardized glossary. For more information, please refer to "Standard Glossary of Terms used in Software Testing" published by ISTQB
