System Settings

General Settings

In general setting, you can configure system behaviors including:

Where and how to store application logs

Display language used in Hummingbird

Please note that the display language is applied to Hummingbird application itself. Messages from Test Framework is always English, and message from the system depends on your operating system configuration. You need to restart Hummingbird Application before the change on display language are effective.

Your preferred theme color

You can choose from 10 predefined color schemes. Change is taken account immediately.

Font size

Hummingbird is already a DPI-Aware application, it adapts automatically to your DPI settings. If you want some additional zoom on high resolution display adapters, you can increase font size by choosing Large.  Change is taken account immediately.

 

Test Settings

Here you can changes the parameter for Hummingbird Test Framework itself, the parameters for all services, System Variables, Customer Variables, Test Environments and Security Tokens.

Test Framework Settings

Max Saved requests per service: Hummingbird application saves every request to edit unless there are more different request than this parameter indicated. For example, if you tested a service with 20 different requests, the last 15 request will be stored and can be switched

Service settings

In this section you can change the configuration of service simulation services, such as adjust the TCP port used to expose simulated services, queue length, maximum connections, etc. for ordinary usage, you don’t need to change these settings. But when you planned to run performance test, you may need to increase max queue length and thread count to increase the maximum connection could be made.

System variables

System variables are variables declared in simulated services, usually they are environment dependent values such as service Host Name, Port, Uri, Path or other tokens. You need to set these variables for each test environment declared.

Custom variables

Custom variables are defined by user and can be used to replace tokens in Requests, Replies and some automated test steps. Values of custom variable are also environment dependent, which means you need to assign a value for each test environment.

Security tokens

Security tokens stores the user name password used to authenticate yourself when calling a service. For example, for those service protected by Basic Authentication and Digest Authentication.

If the service uses Windows Integrated Authentication, NTLM or Kerberos, you don’t need to set credentials here, these kind of authentication uses your Windows Identity automatically. Make sure your windows identity have access to the target service.

All test settings will be stored in a Profile, which you can export for another user or embedded in an automated test project.

Define Test Environments

In test settings, you are able to define one or more test environments. For each test environment, set values for system variables, custom variables and credentials.
